Nightcrawler

A web application that collects and analyzes news articles from different countries.

Nightcrawler is a simple experimental website that I decided to make so to demonstrate & practice my ability in backend programming. Also, I'm interested in how different countries react to each other through the news articles.

Nightcrawler uses Python libraries, such as BeautifulSoup and NLTK, to crawl and analyze news articles from different news publishers, each representing different countries. The nytimes represents the New York Times (USA); the ecns represents English-language website of China News Service (China); the yonhap represents Yonhap News Agency (South Korea); and the japantimes represents The Japan Times (Japan). The analysis will be done on news articles to extract the information about its top 10 frequent words and sentiment scores to the PostgreSQL database. The sentiment scores are scores in range from -1 to 1, where the highest score represents the most positive words (most positive opinion) toward the news article subject.

In the Analysis tab , the user can check the current international state of affairs by finding keywords (name) of other countries in the selected country. I even took one more step by adding a database search ability, which the user can type their own keywords to check the database. The data is presented in a beautiful charts with help of a python library named HighCharts. Things I've learned from this project: Since it was my first personal project as well as my first Python project, I've learned a lot about Python and Django, as well as many Python libraries, such as BeautifulSoup, NLTK and HighCharts. I'ved learned a great deal about the MVC (Model-View-Controller) design pattern from utilizing Django framework. Django follows the traditional MVC (although with different terminology), and I've realized it is very crucial to deisgn MVC carefully; I've used two types of models in the application: one for storing sentiment analysis and top 10 frequent top words data, and another one for higher database level to store the combinatory information for the internatonal state of affairs. However, I've later found out, the second type of model was not necessarily since I could've just implement an additional Controller (type and search) function for the purpose. Additionally, since the news publisher publishes news articles in different timezone and Django doesn't implement timezone by default; it was an interesting lesson to unite the timezone to UTC 0.

Overall, it was a very fun project for me to start learning about Python and Backend programming! I would love to use these knowledge on any future projects if possible.

*Fun Fact* I was REALLY hoping to add North Korea publisher into the project but I have to abandon that idea due to low and slow response on North Korea news publisher websites.
