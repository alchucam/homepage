Piano Simulator

A MIPS program for creating a fun learning experience

Piano Simulator is a MIPS program that was written as part of the school project during my first year of learning programming. The project assignment was a no-limit free assignment but I decided to make this program as it provides experience for basic MVC design pattern

Using MARS, an GUI-based MIPS emulator, I've written the Piano Simulator with MIPS, an assembly language. The program is meant to simulate the real piano keys, and helps user to get familiarized with piano. The program contains musical notes from middle-C to upper/high-F including all the black and white keys in between; essentially, making it a 18-key keyboard. From the computer keyboard row from q to [ represents the black keys in real piano. For example, keyboard q represents C in piano; meanwhile, keyboard 9 represents high-C sharp in piano.

The user can record their play in 'Keyboard and Display MMIO Simulator' and save it to a designated file, musicalNotes.ntxt. In the 'Bitmap Display', the notes will fall from the top; the user needs to hit the matching keyboard key to match the falling notes as it hits the line at the bottom. If the user is able to hit the note at the right time, the mark of the line will flash as pink color; if not, it will flash as red color. Additionally, the user can change the volume and instruments in settings.

Piano Simulator helped me understand the basic MVC design pattern, where I've thought about how the user can record and save their plays into a textfile, and I've also needed to think about how data should be stored, so that it will be loaded to the program in a real time smoothly regardless of user's input. Furthermore, I had to think about how to connect these save/load feature with a color change feature which depends on hitting the falling nodes at the right time. Funny enough, what gave me hard time was not the programming itself, but the lack of preparing MVC design pattern beforehand. However, once I set every roadmap clear, I breezed through (although a relatively long coding amount) the project quickly.
